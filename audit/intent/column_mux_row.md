# T1.1-A — Intent doc: production `mux_<tag>` (ColumnMuxRow)

## What this cell IS electrically

An `M:1` column multiplexer per output bit. For an array of `bits × mux_ratio` physical column pairs, `M = mux_ratio` of those pairs share each output bit-line pair via NMOS pass-gates selected by `col_sel_<k>`. Total devices: `2 × bits × mux_ratio` NMOS pass-gates (one per BL leg + one per BR leg per physical column pair).

For `sram_weight_bank_small` (32 bits × mux 4) → 256 NMOS; for `sram_activation_bank` (64 bits × mux 2) → 256 NMOS. Same device count by coincidence.

**Per-pair structure** (for column pair `i`, where `bit = i // mux_ratio`, `k = i % mux_ratio`):
- `MN_BL`: drain=`bl_<i>`, source=`muxed_bl_<bit>`, gate=`col_sel_<k>`, body=`VNB`
- `MN_BR`: drain=`br_<i>`, source=`muxed_br_<bit>`, gate=`col_sel_<k>`, body=`VNB`

Both NMOS w=0.42 µm, l=0.15 µm.

## Hand-written subckt body (paraphrased)

```
.subckt mux_<tag>
+ col_sel_0 col_sel_1 ... col_sel_{M-1}
+ bl_0 br_0 bl_1 br_1 ... bl_{N-1} br_{N-1}
+ muxed_bl_0 muxed_br_0 ... muxed_bl_{bits-1} muxed_br_{bits-1}
+ VNB
* For each i in 0..N-1:  bit = i//M, k = i%M
X_mux_bl_<i>  bl_<i>  col_sel_<k>  muxed_bl_<bit>  VNB  sky130_fd_pr__nfet_01v8  w=0.42 l=0.15
X_mux_br_<i>  br_<i>  col_sel_<k>  muxed_br_<bit>  VNB  sky130_fd_pr__nfet_01v8  w=0.42 l=0.15
.ends
```

Total: 2N NMOS, all identical sizing.

## How rekolektion uses it

- Placed below the bitcell array; receives `bl_<i>`/`br_<i>` from the array, drives `muxed_bl_<bit>`/`muxed_br_<bit>` toward sense / write driver.
- `col_sel_<k>` is generated by the predecoder from the column-address bits.
- Strap-aware (Option B refactor / task #73): for mux variants where pass-through columns are inserted at strap positions, the `bl_<i>`/`br_<i>` index follows the strap-aware physical column order.

## Severity

- **PASS at intent level.** Hand-written body matches layout topology and devices.

## Ambiguities / followups

- The mapping `bit = i // mux_ratio` assumes column pairs are *contiguous* under each mux output. If a future column-mux scheme interleaves bits (e.g. for write-disturb mitigation), this map needs to change in lockstep with the layout's `bl_<i>` order.
- VNB bias: NMOS bodies tied to the chip-wide PSUB, equated to VGND by netgen at the macro level. No metal-contact concern; PSUB is the substrate.
