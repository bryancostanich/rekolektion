# T4.1 — Intent doc: `sky130_cim_supercell` (production CIM unit cell)

## What this cell IS electrically

Production CIM unit cell. Wraps the foundry `sky130_fd_bd_sram__sram_sp_cell_opt1` 6T bitcell (with a Q-tap modification exposing the storage node) plus 1 NMOS pass transistor T7 (gate=`MWL`, source=`Q`, drain=cap_bot) and 1 MIM capacitor C_C (`MBL`–cap_bot). Topology: foundry 6T → T7 → MIM cap → MBL.

**Variant configurations** (`CIM_SUPERCELL_VARIANTS` in `src/rekolektion/bitcell/sky130_cim_supercell.py`):

| Variant | cap_w (µm) | cap_l (µm) | C (fF) | rows | cols |
|---------|-----------|-----------|--------|------|------|
| SRAM-A  | 1.30      | 3.10      | 8.1    | 256  | 64   |
| SRAM-B  | 1.10      | 2.65      | 5.8    | 256  | 64   |
| SRAM-C  | 1.10      | 1.80      | 4.0    | 64   | 64   |
| SRAM-D  | 1.00      | 1.45      | 2.9    | 64   | 64   |

**Declared port list** (from `_write_supercell` in `src/rekolektion/macro/cim_spice_generator.py`):
`BL BR WL MWL MBL VPWR VGND VPB VNB` — 9 ports.

```

Xfoundry BL BR VGND VPWR VPB WL VNB Q sky130_fd_bd_sram__sram_sp_cell_opt1_qtap
XT7 cap_bot MWL Q VNB sky130_fd_pr__nfet_01v8 w=0.42 l=0.15
```

## How rekolektion uses it

- `cim_supercell_array.py` instantiates the supercell N×N times to form the bitcell array (`cim_array_sram_d_64x64`).
- `cim_assembler.py` places the array + MWL drivers + MBL precharges + MBL sense rows into a complete macro and emits `output/cim_macros/cim_sram_<v>_<r>x<c>/<top>.gds`.
- The `.sp` reference netlist is generated by `cim_spice_generator.generate_cim_reference_spice()`.

## Severity flags

- **P1 — VPB/VNB ports passed VPWR/VGND at instance.** The supercell .subckt declares VPB and VNB as separate body-bias terminals (good intent) but every callsite in the array short-circuits them to VPWR/VGND. **The body-bias terminals are not exercised in the reference netlist; LVS clean is conditional on netgen equating them.** This is the same label-merge fake the T4.4 audit found.
- **P1 — `BR` vs `BLB` name drift.** Supercell generator uses `BR`; extracted CIM variant subckts use `BLB`. Mismatched names across the toolchain are fragile — confirm any LVS rename rule that bridges them.
- **N-A for foundry transistor count.** The foundry `sky130_fd_bd_sram__sram_sp_cell_opt1.subckt.sp` (8 transistors with auto-named internal nodes `a_38_212#`, `a_174_54#`, …) was not generated by rekolektion and is not subject to T4.1 generator-vs-body diff.

## Ambiguities

- The supercell builder forwards the foundry cell's existing pin labels (BL/BR/VGND/VPWR/VPB/VNB) by reference rather than re-stamping them at supercell scope (comment at `sky130_cim_supercell.py:~Forward foundry cell's external pin labels`). Magic flat extraction depends on those labels being visible at the supercell boundary; if any are stripped during reference flattening (cf. F39 — known fact at the supercell array level for BL/BR), they fail to promote as supercell ports. Need T2 connectivity layer-trace for confirmation, which already produced **issue #7** for BL/BR. The supercell layout root-cause for #7 lives in this file's geometry.
